# Skill: wf-summarise

セッション引き継ぎ担当。コンテキスト劣化時・失敗蓄積時に使用。

## 使用条件

応答が混乱・繰り返し / 同じ問題で3回以上失敗 / 長いセッションを終了する前

## プロセス

1. セッション内容を検出（何をしていたか）
2. 状態収集（ブランチ・ファイル・成果物）
3. 適切な場所に要約を保存:
   実装中→タスクファイル（`.agents/tasks/<issue>/task-N.md`）の Implementation Notes
   設計中・調査中・不明→**Issue のコメント**に記録する
4. 新セッション用の引き継ぎプロンプトを提供

## 要約形式

```
## [フェーズ] 引き継ぎ - [timestamp]
ステータス: IN_PROGRESS|BLOCKED|PARTIAL_SUCCESS
作業内容: / 完了: / 未完了: / 試みたこと(失敗時): / ブロッカー: / 推奨次ステップ:
```

## 3回失敗の分析

```
試みたこと: / 失敗理由: / 仮説: コンテキスト汚染|AIの盲点|誤アプローチ
推奨: 新セッション | 手動制御 | 別のアプローチ
```

## 引き継ぎプロンプト形式

```
[フェーズ]を続ける。[Issue/タスク]: [参照] / 引き継ぎ: [ファイルパス]
前のセッション[ステータス]。続行前に引き継ぎノートを確認。
```

## ルール

要約後は作業を続けない / コミットしない / 失敗を正直に記録 / 要約は50行以内

## 完了後

```
要約完了: [保存先] / 人間: 確認→新セッション→引き継ぎプロンプト貼り付け
```
